hexo博客建站指南

有很多人选择把博客建在Github上，这是一个开源代码托管网站，我也把我的博客建在了GitHub上。当初我还为了用哪个博客框架而烦恼呢，因为有很多有名的开源博客框架，比如Hexo、WordPress、VuePress、Halo 等，wordpress是动态博客框架所以不能建在GitHub上，相比较剩下三种，我选择了Hexo。

Hexo是基于NodeJs的静态博客框架，简单、轻量，它生成的静态站点可以托管在Github和Gitee上。Hexo还支持MarkDown语法，是很多程序猿的福音，它可以一键部署，支持超多插件，拥有超多主题，是个不错的选择。

前置条件—环境搭建

一、安装nodejs

因为hexo是基于nodejs，所以需要安装

  • 打开nodejs官网，安装长期稳定版
  • 打开安装程序，安装选项全部默认
  • 打开cmd，输入node -v和npm -v,输出版本号即代表安装成功

二、安装git

  • 打开git安装地址，点击Download下载对应版本

  • 打开安装程序，可选全部默认，或者按需修改，我自己修改了安装地址到 非c盘位置

  • 安装完成后再桌面右键，出现Git GUI Here和Git Bash Here后代表安装成功，或者可以打开cmd 输入 git -v，出现版本号代表安装成功

  • 配置git的邮箱和用户名

三、注册github账号

由于需要将博客挂载到github站点上，所以需要先注册账号

打开GIthub官网，点击 按钮注册账号，然后输入邮箱、密码、用户名，然后创建

安装hexo及配置

初始化项目

  • 在本地合适位置新建文件夹，用来存放自己的博客
  • 在博客文件夹中右键点击Git Bash Here，调出git命令窗口，输入npm install hexo-cli -g命令下载hexo-cli，并设置全局生效
  • 安装完成后输入命令hexo -v查看，如果出现版本号代表安装成功
  • 接下来开始构建初始文件夹，输入命令hexo init blog,此处’blog’为自定义博客名称
  • 输入cd blog命令进入blog文件夹内，输入npm install下载项目所需依赖

    这样就会将这个主题应用到自己的项目中了，接下来就是将主题里面的文件夹进行配置，找到theme文件夹下hexo-theme-linear主题文件夹，再打开主题的配置文件 _config.yml

  • 然后打开根目录，也就是blog目录下的_config.yml文件，用于配置博客的标题和github地址推送
